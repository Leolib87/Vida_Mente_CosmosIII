Análisis de Metagenomas

Estrategia y guía práctica para la identificación de genes de interés en datos genómicos ambientales.

Acceso y Herramientas

El análisis se realizará de forma remota en un servidor computacional. A continuación se detallan las herramientas necesarias para establecer la conexión y gestionar los archivos.

  • Cliente SSH: Para la conexión segura al servidor.
    • Herramienta recomendada: PuTTY
    • Puerto de conexión: 4603
  • Transferencia de Archivos (SFTP): Para mover archivos entre el computador local y el servidor.
    • Herramientas recomendadas: FileZilla o WinSCP.
  • Software de Análisis Principal: La herramienta clave para la búsqueda de homología de secuencias.
    • Software: DIAMOND. Se utilizará para realizar alineamientos de tipo blastp a alta velocidad.

Descripción de los Datos

Los datos de partida son una colección de metagenomas obtenidos de la base de datos pública MG-RAST. Es fundamental comprender su estructura y particularidades antes de iniciar el análisis.

  • Ubicación en el servidor: /meet-meet/metagenomas/. Dentro, se organizan en subcarpetas por ambiente (ej: marino/).
  • Formato de Archivo: .faa (FASTA de aminoácidos).
    Decisión Estratégica: Se trabaja con secuencias de aminoácidos en lugar de nucleótidos para aprovechar la velocidad de DIAMOND, que es miles de veces más rápido que BLAST tradicional para búsquedas de proteína contra proteína.
  • Nomenclatura: Siguen el patrón MGM[ID_Muestra].550.2.faa.
    • MGM: Indica que es un Metagenoma.
    • [ID_Muestra]: Identificador único de la muestra en MG-RAST, permite buscar sus metadatos (ej: coordenadas geográficas).
    • .550.2: Indica que los datos han pasado por una etapa de control de calidad y filtrado inicial en la plataforma de origen.
  • Problema Crítico Detectado: Existe una enorme variabilidad en el tamaño de los metagenomas. Muchos archivos contienen muy pocas secuencias (ej. 600-900), lo que los hace estadísticamente poco informativos. El primer paso del análisis será filtrar estas muestras.

  1. Inventario y Filtrado Inicial de Datos

    El primer paso es crucial: debemos limpiar nuestro set de datos para trabajar solo con muestras informativas. No modificaremos los datos originales.

    1.1. Crear un directorio de trabajo y copiar los datos:

    # Navega al directorio principal y crea una carpeta para el análisis cd /meet-meet/ mkdir analisis_dsr # Copia los datos crudos para no alterarlos (esto puede tardar) cp -r metagenomas/marino/ analisis_dsr/

    1.2. Contar secuencias y generar un reporte:

    Un archivo FASTA de proteínas tiene 2 líneas por secuencia (header y secuencia). Contaremos las líneas totales y dividiremos por 2 para obtener el número de secuencias.

    # Entra al directorio de trabajo cd analisis_dsr/marino # Cuenta las líneas de todos los archivos .faa y guarda el resultado wc -l *.faa > ../conteo_lineas.txt

    1.3. Procesar el reporte y definir el umbral:

    Descarga el archivo conteo_lineas.txt a tu computador. Usa una hoja de cálculo para dividir el número de líneas por 2 y obtener el total de secuencias por muestra. Ordena los metagenomas por tamaño y analiza la distribución. Define un umbral razonable (ej. > 50,000 secuencias) basado en la literatura o en la distribución de tus datos para descartar los archivos más pequeños.

  2. Construcción de la Base de Datos de Referencia (DSR)

    Necesitamos una base de datos local con las secuencias del gen de interés (ej. DSR) contra la que compararemos nuestros metagenomas.

    2.1. Obtener secuencias de referencia:

    Ve a bases de datos como UniProt o NCBI Protein. Busca "dissimilatory sulfite reductase" (o genes específicos como dsrA y dsrB). Descarga una selección representativa de secuencias en formato FASTA.

    2.2. Unificar y crear la base de datos DIAMOND:

    # Sube tus archivos FASTA (ej. dsrA.fasta, dsrB.fasta) a tu directorio de análisis # Combínalos en un solo archivo cat dsrA.fasta dsrB.fasta > dsr_database.faa # Crea la base de datos para DIAMOND # --in: archivo de entrada FASTA # -d: nombre para la base de datos de salida (.dmnd) diamond makedb --in dsr_database.faa -d dsr_database
  3. Validación del Método con un Metagenoma "Mock"
    Paso de Control de Calidad: Antes de procesar cientos de archivos, nos aseguramos de que el método funciona creando una prueba controlada para detectar las secuencias que sabemos que están ahí y evitar falsos positivos.

    3.1. Crear el metagenoma "mock":

    Toma un metagenoma real (uno de los filtrados) y añádele manualmente unas 5-10 secuencias de tu archivo dsr_database.faa. Guárdalo como mock_metagenome.faa. Ahora sabes exactamente cuántos hits "reales" deberías encontrar.

    3.2. Ejecutar DIAMOND y verificar:

    # Ejecuta la búsqueda de DIAMOND # -d: tu base de datos | -q: tu archivo de consulta (el mock) # -o: el archivo de salida | --outfmt 6: formato tabular # --sensitive: modo más lento pero más preciso diamond blastp -d dsr_database.dmnd -q mock_metagenome.faa -o mock_results.tsv --outfmt 6 --sensitive

    Revisa el archivo mock_results.tsv. Deberías encontrar los hits correspondientes a las secuencias que insertaste. ¡Si es así, el método es confiable!

  4. Procesamiento a Gran Escala

    Ahora aplicamos el método validado a todos nuestros datos filtrados, tanto para el gen de interés como para un gen de normalización.

    4.1. Crear base de datos para normalización:

    Repite el paso 2, pero esta vez con un gen housekeeping como rpoB para poder normalizar los resultados. Llama a la base de datos rpob_database.

    4.2. Automatizar con un script:

    Crea un archivo llamado run_analysis.sh. Este script recorrerá cada metagenoma filtrado y ejecutará DIAMOND para DSR y rpoB.

    #!/bin/bash # Script para procesar todos los metagenomas # Crea directorios para los resultados mkdir -p results_dsr mkdir -p results_rpob # Itera sobre cada archivo .faa en el directorio de trabajo for metagenome in *.faa do # Extrae el nombre base del archivo base=$(basename ${metagenome} .faa) echo "Procesando ${base}..." # Ejecuta DIAMOND para DSR (gen de interés) diamond blastp -d ../dsr_database.dmnd -q ${metagenome} -o ../results_dsr/${base}.tsv --outfmt 6 --sensitive # Ejecuta DIAMOND para rpoB (gen de normalización) diamond blastp -d ../rpob_database.dmnd -q ${metagenome} -o ../results_rpob/${base}.tsv --outfmt 6 --sensitive done echo "Análisis completo."

    Ejecuta el script desde el directorio analisis_dsr/marino/ con: bash ../run_analysis.sh (si el script está en la carpeta superior).

  5. Análisis y Visualización Final

    El último paso es consolidar los resultados, calcular la abundancia relativa y presentarlos de forma visual e intuitiva.

    5.1. Extraer y normalizar los conteos:

    Para cada muestra, cuenta el número de líneas (hits) en su archivo de resultados DSR y rpoB. Luego, calcula la abundancia relativa: (Total hits DSR) / (Total hits rpoB). Esto se puede hacer con un script (ej. Python/Pandas) o con comandos de bash.

    5.2. Georreferenciación y Mapeo:

    Para cada ID de muestra (ej. MGM4440039), búscalo en MG-RAST para encontrar sus metadatos, específicamente las coordenadas (latitud y longitud). Finalmente, usa una herramienta de visualización (como Python con Folium/Geopandas, o QGIS) para crear un mapa del mundo donde cada punto represente una muestra, y el color o tamaño del punto indique la abundancia relativa de genes DSR.
